# Specification: python-gaze-integration

## ADDED Requirements

### FR-PGI-001: Language-conditional tool dispatch in `uf init`

`uf init` MUST dispatch tool initialization based on the detected
project language. When a `simpleTool` entry carries a non-empty
`forLang` value, `uf init` MUST skip that tool if the detected
project language does not match `forLang`. When `forLang` is empty,
the tool MUST run for all projects (existing behavior).

#### Scenario: Python project with gazepy in PATH

- **GIVEN** a project directory containing `pyproject.toml`
- **AND** `gazepy` is available in PATH
- **WHEN** `uf init` runs
- **THEN** `gazepy init` is called and `.opencode/agents/gaze-reporter.md`
  is written with Python-oriented content

#### Scenario: Go project with gazepy in PATH

- **GIVEN** a project directory containing `go.mod`
- **AND** `gazepy` is available in PATH
- **WHEN** `uf init` runs
- **THEN** `gazepy init` is NOT called (language mismatch)
- **AND** `gaze init` IS called if `gaze` is in PATH

#### Scenario: Python project with neither tool in PATH

- **GIVEN** a project directory containing `pyproject.toml`
- **AND** neither `gaze` nor `gazepy` is available in PATH
- **WHEN** `uf init` runs
- **THEN** both tool init steps are skipped silently
- **AND** `uf init` completes successfully

### FR-PGI-002: `gazepy` install in `uf setup`

`uf setup` MUST attempt to install `gazepy` via
`brew install unbound-force/tap/gazepy` when Homebrew is available
and `gazepy` is not already in PATH. When Homebrew is unavailable,
`uf setup` MUST skip the step and emit a download hint pointing to
the GitHub releases page. The `setup.tools.gazepy.method` config key
MUST be respected to override install behavior.

#### Scenario: Homebrew available, gazepy not installed

- **GIVEN** Homebrew is available on the developer machine
- **AND** `gazepy` is not in PATH
- **WHEN** `uf setup` runs
- **THEN** `brew install unbound-force/tap/gazepy` is executed
- **AND** the result is reported as `installed: via Homebrew`

#### Scenario: Homebrew unavailable

- **GIVEN** Homebrew is not available on the developer machine
- **AND** `gazepy` is not in PATH
- **WHEN** `uf setup` runs
- **THEN** the step is skipped with a download hint
- **AND** `uf setup` continues and completes successfully

#### Scenario: gazepy already installed

- **GIVEN** `gazepy` is already in PATH
- **WHEN** `uf setup` runs
- **THEN** the install step is skipped
- **AND** the result is reported as `already installed`

#### Scenario: Config method override — skip

- **GIVEN** `.uf/config.yaml` contains `setup.tools.gazepy.method: skip`
- **WHEN** `uf setup` runs
- **THEN** the `gazepy` install step is skipped entirely

---

### FR-PGI-003: `gazepy` doctor check

`uf doctor` MUST include `gazepy` as a recommended (non-required)
tool check. The severity MAY be overridden to `required` or
`optional` via `doctor.tools.gazepy` in `.uf/config.yaml`.

#### Scenario: gazepy missing, default severity

- **GIVEN** `gazepy` is not in PATH
- **AND** no `doctor.tools.gazepy` override is configured
- **WHEN** `uf doctor` runs
- **THEN** the `gazepy` check is reported as missing with
  `recommended` severity

---

### FR-PGI-004: `gazepy` documented in config template

The `.uf/config.yaml` template generated by `uf init` MUST document
the `setup.tools.gazepy` key with valid method values
(`auto`, `homebrew`, `skip`) in commented-out form.

#### Scenario: New project config template

- **GIVEN** a new project where `uf init` has not been run
- **WHEN** `uf init` runs and generates `.uf/config.yaml`
- **THEN** the generated file includes a commented-out
  `gazepy` entry under `setup.tools`

## MODIFIED Requirements

### `simpleTool{}` struct (scaffold.go)

The `simpleTool` struct MUST include a `forLang string` field.
When `forLang` is empty, behavior is unchanged from the existing
implementation. Previously: struct had no language filter field.
